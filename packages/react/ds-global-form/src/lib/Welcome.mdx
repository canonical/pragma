# Form Components — React

Forms are where users provide data — getting them right is critical for both usability and data quality. This package provides a `Form` wrapper, a `Field` router, 15 input types, a middleware system, and accessible form structures that work consistently across all Canonical products.

## Architecture

The form system has four layers:

```
Form            react-hook-form provider + subgrid container
  └─ Field      type-safe router — selects input by inputType prop
       └─ Wrapper    label, description, error, ARIA, middleware
            └─ Input     one of 15 built-in types or a custom component
```

`Form` wraps children in a `FormProvider` and renders as a CSS subgrid, inheriting columns from the parent `.grid` context. `Field` is a discriminated union — the `inputType` prop selects which input component renders, and TypeScript enforces the correct props for that type. `Wrapper` composes the field chrome (label, description, error message, ARIA attributes) and applies middleware. The input component handles the actual user interaction.

## Quick example

```tsx
import { Form, Field } from "@canonical/react-ds-global-form";

<div className="grid responsive">
  <Form onSubmit={(data) => console.log(data)}>
    <Field name="name" inputType="text" label="Name" />
    <Field
      name="email"
      inputType="text"
      label="Email"
      registerProps={{ required: "Email is required" }}
    />
    <Field
      name="role"
      inputType="select"
      label="Role"
      options={[
        { value: "dev", label: "Developer" },
        { value: "design", label: "Designer" },
      ]}
    />
    <button type="submit">Submit</button>
  </Form>
</div>
```

Forms require a `.grid` parent — `Form` renders as a subgrid and inherits the column structure. See [Getting Started](?path=/docs/global-form-getting-started--docs) for the full walkthrough.

## Validation

Validation integrates with [react-hook-form](https://react-hook-form.com/) through the [`registerProps`](https://react-hook-form.com/docs/useform/register) prop. Error states apply a `.danger` CSS class that shifts the input border to the error colour. See [Getting Started](?path=/docs/global-form-getting-started--docs) for details.

## Middleware

Middleware wraps input components to add cross-cutting behaviour — fetching options from an API, server-side validation, conditional display. See [Creating Middleware](?path=/docs/global-form-guides-creating-middleware--docs).

## Installation

```bash
bun add @canonical/react-ds-global-form
```

Requires `@canonical/react-ds-global` and `@canonical/styles` as peer dependencies.
