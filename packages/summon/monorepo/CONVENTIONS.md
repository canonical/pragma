# Monorepo Conventions

> Generated by `@canonical/summon-monorepo` v0.1.0
> Last updated: 2026-03-08

## Taxonomy Key

| Prefix | Domain | Description |
|--------|--------|-------------|
| `M1` | Runtime | Package manager and runtime constraints |
| `M2` | Orchestration | Task running, caching, dependency ordering |
| `M3` | Config | Shared TypeScript, Biome, architecture configs |
| `M4` | Testing | Test runner, naming, coverage |
| `M5` | Build | Module format, output structure |
| `M6` | Scripts | Root-level npm scripts |
| `M7` | CI | Workflow files and pipeline structure |
| `M8` | Release | Versioning, changelogs, publishing |
| `M9` | Git | Commit format, merge strategy, branch protection |
| `M10` | Dependencies | Renovate configuration and grouping |
| `M11` | Layout | Repository file organization |
| `M12` | Metadata | Required package.json fields |

---

## M1 — Runtime

| ID | Rule | Gate |
|----|------|------|
| M1.1 | Package manager is **Bun** | `bun.lock` exists |
| M1.2 | Bun version is pinned in `.github/actions/setup-env/action.yml` | version pinned |
| M1.3 | `bun.lock` is committed (not gitignored) | file tracked by git |
| M1.4 | CI uses `bun install --frozen-lockfile` | frozen-lockfile in CI |

---

## M2 — Orchestration

| ID | Rule | Gate |
|----|------|------|
| M2.1 | **Lerna** handles workspace task execution and versioning | lerna.json exists |
| M2.2 | **Nx** provides caching and dependency-aware task ordering | nx.json exists |
| M2.3 | Root scripts use `lerna run <script>`, never `bun run --filter` | scripts use lerna |
| M2.4 | Fixed versioning: all packages share one version number | lerna.json version = fixed |

---

## M3 — Config

| ID | Rule | Gate |
|----|------|------|
| M3.1 | TypeScript config consumed from `@canonical/typescript-config-*` npm package | extends npm package |
| M3.2 | Biome config consumed from `@canonical/biome-config` npm package | extends npm package |
| M3.3 | Packages extend npm packages directly, not relative root paths | no `../../` in extends |
| M3.4 | Architecture validated by `@canonical/webarchitect` with appropriate ruleset | webarchitect configured |
| M3.5 | Root `tsconfig.json` extends the chosen shared config package | extends field present |
| M3.6 | Package `tsconfig.json` extends root or appropriate variant | extends field present |
| M3.7 | `tsconfig.build.json` used for production builds | file exists for library pkgs |
| M3.8 | Build output: `dist/esm/` for JS, `dist/types/` for declarations | output dirs correct |
| M3.9 | `declarationMap: true` and `sourceMap: true` enabled | flags set |
| M3.10 | Root `biome.json` extends `@canonical/biome-config` with repo-specific scope | extends + scope |
| M3.11 | Package `biome.json` extends `@canonical/biome-config` directly | extends set |
| M3.12 | Quote style: double quotes | biome default |
| M3.13 | Indent: 2 spaces | biome default |
| M3.14 | Semicolons: always | biome default |

## M4 — Testing

| ID | Rule | Gate |
|----|------|------|
| M4.1 | Test runner is **Vitest** | vitest in root deps |
| M4.2 | Test file naming: `*.tests.ts` (plural, not `*.test.ts`) | no singular test files |
| M4.3 | Test files live alongside source in `src/` | test files under src/ |

| M4.6 | Vitest include pattern: `src/**/*.tests.ts` | include pattern set |

## M10 — Dependencies

| ID | Rule | Gate |
|----|------|------|
| M10.1 | **Renovate** manages automated dependency updates | `renovate.json` exists |
| M10.2 | Schedule: weekly (before 9am on Monday) | schedule configured |
| M10.3 | Batching families group related dependencies into single PRs | groups configured |
| M10.4 | Automerge: patch and minor updates for devDependencies | automerge rules set |
| M10.5 | Labels: `Maintenance` applied to all Renovate PRs | label configured |
| M10.6 | Commit format: conventional commits (`chore(deps): ...`) | format set |

### Renovate Batching Families

| Group | Packages |
|-------|----------|
| `biome` | `@biomejs/biome`, `@canonical/biome-config` |
| `typescript` | `typescript`, `@canonical/typescript-config-*` |
| `vitest` | `vitest`, `@vitest/*` |
| `lerna-nx` | `lerna`, `nx`, `@nx/*` |
| `canonical` | `@canonical/*` (catch-all for internal packages) |

Domain-specific groups (e.g., `lit`, `relay`, `storybook`, `react`) are added per-repo as needed.

## M11 — Layout

| ID | Rule | Gate |
|----|------|------|
| M11.1 | CI workflows in `.github/` | directory exists |
| M11.2 | All workspace packages in `packages/` | directory exists |
| M11.3 | Monorepo-level scripts in `scripts/` | directory exists |
| M11.4 | Root config files: `biome.json`, `lerna.json`, `nx.json`, `tsconfig.json` | files exist |
| M11.5 | `.gitignore` organized by category | file exists |

### Repository Layout

```
<repo>/
├── .github/           # M11.1  CI workflows and actions
├── packages/          # M11.2  All workspace packages
├── scripts/           # M11.3  Monorepo-level scripts
├── biome.json         # M11.4  Root Biome config
├── lerna.json         # M11.4  Lerna config (fixed version)
├── nx.json            # M11.4  Nx caching config
├── package.json       # M11.4  Root workspace config
├── renovate.json      # M10.1  Renovate config
├── tsconfig.json      # M11.4  Root TypeScript config
└── .gitignore         # M11.5  Organized by category
```

## Quick Gate Checklist

Use this as a pass/fail gate for new monorepo setup:

```
[ ] M1.1   Bun is the package manager
[ ] M1.2   Bun version pinned in CI
[ ] M2.1   Lerna configured for task execution
[ ] M2.2   Nx configured for caching
[ ] M2.4   Fixed versioning enabled
[ ] M3.1   TypeScript config from @canonical/typescript-config-*
[ ] M3.2   Biome config from @canonical/biome-config
[ ] M3.3   No relative path extends in packages
[ ] M4.1   Vitest is the test runner
[ ] M4.2   Test files use *.tests.ts (plural)
[ ] M5.1   ESM module format ("type": "module")
[ ] M7.1   CI workflow runs on push + PRs
[ ] M8.1   Lerna fixed versioning
[ ] M9.1   Conventional commits enforced
[ ] M10.1  Renovate configured
[ ] M12.1  Package names scoped under @canonical/
```
