# Package Conventions

> Generated by `@canonical/summon-package` v0.1.0
> Last updated: 2026-03-08

## Taxonomy Key

| Prefix | Domain | Description |
|--------|--------|-------------|
| `P1` | Structure | Package types, directories, entry points |
| `P2` | Naming | Scoping and casing rules |
| `P3` | Config | TypeScript, Biome, build configuration |
| `P4` | Scripts | Required npm scripts |
| `P5` | Testing | Test patterns, runner, coverage |
| `P6` | Metadata | package.json required fields |
| `P7` | Validation | Architecture enforcement |

---

## P1 — Structure

| ID | Rule | Gate |
|----|------|------|
| P1.1 | Every package has exactly one of three types: `tool-ts`, `library`, or `css` | `type ∈ {tool-ts, library, css}` |
| P1.2 | `tool-ts` packages use `src/index.ts` as entry point, no build step | entry = `src/index.ts` |
| P1.3 | `library` packages build to `dist/esm/index.js` via `tsc` | entry = `dist/esm/index.js` |
| P1.4 | `css` packages use `src/index.css` as entry point, no build step | entry = `src/index.css` |
| P1.5 | Source lives in `src/` directory | `src/` exists |
| P1.6 | Tests live in `src/__tests__/` or alongside source in `src/` | test files under `src/` |

### Package Type Matrix

| Type | License | Entry Point | Build | Use Case |
|------|---------|-------------|-------|----------|
| `tool-ts` | GPL-3.0 | `src/index.ts` | None | CLI tools, generators |
| `library` | LGPL-3.0 | `dist/esm/index.js` | `tsc` | Publishable npm packages |
| `css` | LGPL-3.0 | `src/index.css` | None | CSS-only packages |

## P2 — Naming

| ID | Rule | Gate |
|----|------|------|
| P2.1 | Package names are scoped: `@canonical/<name>` | name starts with `@canonical/` |
| P2.2 | Package names use lowercase kebab-case | name matches `/^@canonical\/[a-z][a-z0-9-]*$/` |
| P2.3 | Directory name equals the unscoped package name | dir = name without `@canonical/` |

## P3 — Config

| ID | Rule | Gate |
|----|------|------|
| P3.1 | `tsconfig.json` extends monorepo root or a shared config variant | `extends` field present |
| P3.2 | `tsconfig.build.json` outputs to `dist/esm/` (JS) and `dist/types/` (declarations) | outDir = `dist/esm`, declarationDir = `dist/types` |
| P3.3 | Test files (`*.tests.ts`) are excluded from both tsconfigs | `exclude` includes test pattern |
| P3.4 | React packages have additional `tsconfig.json` with JSX support | jsx field set for React pkgs |
| P3.5 | `biome.json` extends `@canonical/biome-config` (npm package, not relative path) | extends = `@canonical/biome-config` |
| P3.6 | Biome scope includes `["src", "*.json"]` | scope covers src + json |

## P4 — Scripts

| ID | Rule | Gate |
|----|------|------|
| P4.1 | `build` script runs `tsc -p tsconfig.build.json` (library type only) | script exists for library type |
| P4.2 | `build:all` script runs `bun run build` | script exists |
| P4.3 | `check` script runs biome + tsc + webarchitect | script exists |
| P4.4 | `check:fix` runs biome --write + tsc (must NOT include webarchitect) | script exists, no webarchitect |
| P4.5 | `check:biome` runs `biome check` | script exists |
| P4.6 | `check:biome:fix` runs `biome check --write` | script exists |
| P4.7 | `check:ts` runs `tsc --noEmit` | script exists |
| P4.8 | `check:webarchitect` runs `webarchitect <ruleset>` | script exists |
| P4.9 | `test` runs `vitest run` | script exists |
| P4.10 | `test:coverage` runs `vitest run --coverage` | script exists |

---

## P5 — Testing

| ID | Rule | Gate |
|----|------|------|
| P5.1 | Test files use plural naming: `*.tests.ts` | no `*.test.ts` files |
| P5.2 | Tests located in `src/` or `src/__tests__/` | test files under `src/` |
| P5.3 | Test runner is Vitest | `vitest` in devDependencies |
| P5.4 | Coverage uses v8 provider via `@vitest/coverage-v8` | coverage provider = v8 |
| P5.5 | Vitest include pattern covers `src/**/*.tests.ts` | include pattern set |

---

## P6 — Metadata

| ID | Rule | Gate |
|----|------|------|
| P6.1 | `name` field is `@canonical/<name>` | field present and scoped |
| P6.2 | `version` field is managed by Lerna | field present |
| P6.3 | `description` field is non-empty | field present and non-empty |
| P6.4 | `type` field is `"module"` | type = module |
| P6.5 | `license` is `LGPL-3.0` (library/css) or `GPL-3.0` (tool-ts) | license matches type |
| P6.6 | `author` is `{ email: "webteam@canonical.com", name: "Canonical Webteam" }` | field present |
| P6.7 | `repository`, `bugs`, `homepage` fields point to GitHub URLs | fields present |
| P6.8 | Library packages have conditional `exports` map | exports field for library type |

---

## P7 — Validation

| ID | Rule | Gate |
|----|------|------|
| P7.1 | `@canonical/webarchitect` enforces package structure | webarchitect in devDeps |
| P7.2 | Library packages use `library` ruleset | ruleset = library |
| P7.3 | Tool packages use `tool-ts` ruleset | ruleset = tool-ts |
| P7.4 | React packages use `package-react` ruleset | ruleset = package-react |
| P7.5 | Asset packages use `assets` ruleset | ruleset = assets |
| P7.6 | Minimal validation uses `base` ruleset | ruleset = base |

## Quick Gate Checklist

Use this as a pass/fail gate for new packages:

```
[ ] P1.1  Package type declared (tool-ts / library / css)
[ ] P2.1  Name scoped under @canonical/
[ ] P2.2  Name is lowercase kebab-case
[ ] P3.1  tsconfig.json extends shared config
[ ] P3.5  biome.json extends @canonical/biome-config
[ ] P4.3  check script includes biome + tsc + webarchitect
[ ] P4.4  check:fix does NOT include webarchitect
[ ] P5.1  Test files use *.tests.ts (plural)
[ ] P5.3  Vitest is the test runner
[ ] P6.1  Package name is @canonical/<name>
[ ] P6.4  "type": "module" in package.json
[ ] P6.5  License matches package type
[ ] P7.1  webarchitect validates structure
```
