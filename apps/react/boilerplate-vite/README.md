# boilerplate-vite

React application with SSR and routing, generated by `@canonical/summon-application`.

## Development

```bash
# Client-only dev server (no SSR)
bun run dev

# Express dev server with SSR + HMR
bun run dev:express

# Bun dev server with SSR + streaming
bun run dev:bun

# Storybook
bun run storybook
```

## Adding routes

```bash
# Create a new domain
summon domain billing

# Add a route to a domain
summon route billing/invoices

# Create a layout wrapper
summon wrapper sidebar
```

## Structure

```
src/
├── client/         Client entry (hydration)
├── server/         SSR entry + dev servers
├── domains/        Feature domains (routes + pages)
├── lib/            Shared components
├── styles/         CSS
├── assets/         Imported assets (bundled by Vite)
└── routes.tsx      Root route map
public/             Static files served as-is (favicon, robots.txt)
```

> **Note:** this app ships both `src/assets/` (assets imported in code) and `public/`
> (files served by fixed URL). Keeping both by default is provisional — you may remove
> whichever your app doesn't use.
